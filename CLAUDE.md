## Code Architecture and Structure

This is a React application built with Vite and Tailwind CSS. It functions as an "Evolutionary Website Builder" (Webby) that uses the OpenRouter API (via the openai SDK) to generate and iteratively refine website designs through a tournament-style bracket system.

*   **`src/App.tsx`**: The main application component. It manages the state for the different phases of the application:
    *   `setup`: Getting the initial user prompt and enhancing it.
    *   `tournament`: Displaying the generated candidates side-by-side and allowing the user to select a winner. It manages the tournament bracket using a binary tree structure (`BracketNode`).
    *   `result`: Displaying the final winning design with options to copy or download the generated HTML/CSS/JS.
*   **`src/services/openrouter.ts`**: Contains the logic for interacting with the OpenRouter API. It handles:
    *   `enhancePrompt`: Takes the initial user idea and expands it into a detailed prompt. Users can specify any OpenAI-compatible models (e.g., `openai/gpt-5.4` or `google/gemini-3-flash-preview`) for the prompt enhancement and generation steps.
    *   `generateCandidateStream`: Generates the initial design candidates (HTML, CSS, JS, and design philosophy) stream using specific XML-like tags (`<PHILOSOPHY>`, `<HTML>`, etc.).
    *   `mutateCandidateStream`: (Currently unused in App.tsx but available) Mutates a winning candidate into new variations.
*   **`src/i18n.ts`**: Handles internationalization/translations for the application UI.

### Key Concepts

*   **Tournament Bracket**: The core mechanism is presenting two AI-generated design choices to the user. The user's selection advances up a binary tree bracket until a single ultimate winner remains.
*   **Streaming AI Responses**: The application uses the openai SDK to stream responses from OpenRouter to provide real-time feedback as the HTML/CSS/JS is being generated by the model.
*   **Iframe Previews**: The generated HTML, CSS, and JS are rendered safely within `<iframe>` elements to isolate the AI-generated code from the main React application.

## Environment Variables

*   `OPENROUTER_API_KEY`: The API key required to authenticate with the OpenRouter API. This is expected to be provided by the user in the UI or set in the environment (e.g., in a `.env.local` file).
